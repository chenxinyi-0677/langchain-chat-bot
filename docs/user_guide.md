# langchain-chat 用户手册

## 1. 快速上手

### 1.1 启动程序

确保已按照 README 安装依赖并配置好 `.env` 文件后，在终端执行：

```bash
uv run python -m src.main
```

终端会显示程序启动信息，然后自动进入登录流程。

### 1.2 首次使用（创建账号）

第一次启动时，程序检测到还没有用户，会提示：

```
欢迎！请输入用户名创建账号:
```

输入你想要的用户名（例如 `alice`），回车确认。程序会显示：

```
用户 'alice' 创建成功！
当前用户: alice
```

然后你就进入了主菜单，可以看到 `>` 提示符等待输入命令。

### 1.3 登录已有账号

再次启动程序时，会列出已有的用户列表：

```
已有用户:
  1. alice
  2. bob
请输入用户名或编号:
```

你可以输入用户名（如 `alice`）或直接输入前面的编号（如 `1`）来登录。如果输入了一个不存在的用户名，程序会询问是否创建新用户。

## 2. 主菜单命令详解

进入主菜单后，你会看到 `> ` 提示符。输入命令名后回车执行。可以用 Tab 键自动补全命令。

### 2.1 chat —— 开始对话

进入对话前，如果当前还没有活跃会话，程序会让你配置：

1. **选择模型**：输入要使用的模型名，例如 `gpt-4o`、`deepseek-chat`，按回车使用默认值（即 `.env` 里 `MODEL_NAME` 配置的模型）。

2. **选择预设（可选）**：如果有可用预设，会列出供你选择：
   ```
   可用预设:
     0. 不使用预设
     1. 翻译助手 [内置]
     2. 我的预设
   选择预设编号（回车 = 不使用）:
   ```
   输入编号使用对应预设，直接回车则不使用预设。

配置完成后，进入对话模式：

```
对话模式（输入空行返回主菜单）:
你:
```

在 `你:` 后面输入消息，回车发送。AI 的回复会逐字出现在绿色边框面板中，你可以在屏幕上看到文字一点点"长出来"，而不是一次性刷新。

**结束对话**：在 `你:` 提示符下直接按回车（输入空行），就会退出对话返回主菜单。

**查看当前会话信息**：进入对话后，屏幕上方的提示会显示当前模型名。如果想确认当前对话绑定的模型和预设，可以输入 `sessions` 命令查看当前活跃会话的详细信息。

### 2.2 sessions —— 管理历史会话

输入 `sessions` 会列出当前用户的所有历史会话：

```
会话列表
+----+--------------+--------+-------+
| ID | 标题         | 模型   | Token |
+----+--------------+--------+-------+
|  1 | 帮我翻译一下 | gpt-4o |   150 |
|  2 | Python学习   | gpt-4o |   320 |
+----+--------------+--------+-------+
```

- **加载旧会话继续聊**：记下想继续的会话 ID，然后输入 `chat` 命令。如果当前已有活跃会话，会直接进入该会话继续对话；如果没有，新创建的会话会使用你配置的模型。
- **重命名会话**：此功能通过预设管理或直接操作实现（当前 TUI 版本中，会话标题由首条消息自动生成）。
- **删除会话**：同样通过管理界面操作。

> 如果忘记了会话标题，可以使用 `search` 命令按关键词查找。

### 2.3 presets —— 管理预设角色

预设是一段"系统提示词"，用来设定 AI 的角色或行为方式。例如一个"翻译助手"预设可能包含 "Please translate the following text to Chinese" 这样的指令。

输入 `presets` 查看所有可用预设：

```
内置预设
+----+--------------+
| ID | 名称         |
+----+--------------+
|  1 | 翻译助手     |
|  2 | 代码审查     |
+----+--------------+

我的预设
+----+----------+-----------+
| ID | 名称     | 描述      |
+----+----------+-----------+
|  3 | 故事大    | 讲故事    |
+----+----------+-----------+
```

- **内置预设**：系统自带的，所有人都可以查看和使用，但不能编辑或删除。
- **我的预设**：你自己创建的预设，可以编辑、删除。

**创建预设**：通过命令菜单中的创建功能（当前 TUI 版本支持）。

**使用预设**：在 `chat` 命令创建新会话时，可以选择一个预设来设定 AI 角色。

### 2.4 search —— 搜索消息

输入 `search`，然后输入关键词：

```
搜索关键词: Python
```

程序会在当前用户的所有历史消息中搜索，展示匹配的会话和消息内容，方便你快速定位之前的对话。

### 2.5 export —— 导出对话

想把某次对话保存成文件？使用 `export` 命令：

```
要导出的会话 ID: 2
导出成功: data/users/alice/exports/session_2.md
```

导出的文件是 Markdown 格式，可以用任何文本编辑器打开，也可以在支持 Markdown 的笔记软件（如 Obsidian、Notion）中导入。

文件保存在项目目录下的 `data/users/<你的用户名>/exports/` 文件夹里，文件名是 `session_<会话ID>.md`。

### 2.6 switch —— 切换用户

如果你想切换到另一个用户的账号（例如家人共用一个程序）：

```
目标用户名: bob
```

如果 `bob` 不存在，程序会询问是否创建新用户。切换后，所有操作（聊天记录、预设等）都会切换到对应用户的数据。

### 2.7 compare —— 多模型对比

这个功能让你用同一个提示词同时问多个模型，看看各自的回答有什么不同：

```
对比提示词: 用一句话解释什么是递归
模型名（逗号分隔，如 gpt-4o,deepseek-chat）: gpt-4o,deepseek-chat
正在并发调用 2 个模型，请稍候...
```

程序并发请求所有指定模型，并把每个模型的回答分别展示在独立的框里，方便对比。支持显示 token 用量，如果某个模型调用失败也会显示错误信息。

### 2.8 exit —— 退出程序

输入 `exit` 即可退出程序。程序会关闭数据库连接，确保数据安全保存。所有对话数据会自动持久化，下次启动时数据完整保留。

## 3. 常见使用场景

### 3.1 "我想找回上周聊过的一段对话"

如果记得大概内容，用 `search` 命令输入关键词搜索。程序会列出包含该关键词的会话和具体的消息片段，从中找到你想要的会话 ID，然后用 `chat` 进入该会话继续聊。

如果不记得具体内容只记得会话标题，用 `sessions` 命令查看所有会话列表，按标题和模型筛选。

### 3.2 "我想用同一个提示词看不同模型的表现"

用 `compare` 命令。输入同一个提示词，列出多个模型名（用逗号分隔），程序会并发调用所有模型并把结果并排展示。这比分别创建多个会话再手动对比高效得多。

### 3.3 "我想把一次对话保存到本地文件"

用 `export` 命令，输入要导出的会话 ID。文件会以 Markdown 格式保存到项目 `data/users/<用户名>/exports/` 目录下。你可以把这个 `.md` 文件复制到任何地方，用笔记软件或文本编辑器打开。

### 3.4 "我换了电脑，之前的数据怎么迁移"

程序的所有对话数据存储在项目目录下的 `data/sqlite/app.db` 文件（SQLite 数据库）中。迁移步骤如下：

1. **旧电脑**：把 `data/sqlite/app.db` 文件复制出来（整个 `data/` 目录也可以）。
2. **新电脑**：按 README 的步骤重新部署项目（安装依赖、配置 `.env`）。
3. 把备份的 `app.db` 文件放到新电脑项目目录的 `data/sqlite/` 下（如果目录不存在先创建）。
4. 确保新电脑上的用户名和旧电脑一致（因为数据库里按用户名关联用户数据）。

**重要提醒**：这种迁移方式只在新旧两边的项目目录结构一致、用户名相同的前提下有效。如果变更了存储后端（比如从 SQLite 换成 MySQL），需要额外操作。目前没有提供专门的导入/导出工具，数据迁移全靠复制文件。

## 4. 常见问题

### 4.1 输错了命令怎么办？

输错命令不会造成任何损害。程序会显示帮助信息（列出所有可用的命令），然后在 `>` 提示符处等待你重新输入。你可以用 Tab 键自动补全命令名，避免打错。

### 4.2 忘记会话标题，怎么找到想要的会话？

三种方法：
- `sessions` 列出所有会话，从标题和模型名大致判断。
- `search` 按关键词在消息内容中搜索，这是最精确的方法。
- `chat` 随便进入一个会话，输入 `你怎么看` 之类的试探性消息可以帮助你回忆起上下文。

### 4.3 Token 统计是什么意思？

每次对话，程序会记录本次会话累计消耗了多少 prompt tokens（你输入的内容）和 completion tokens（AI 生成的内容）。Token 是 LLM 的计费单位，大致可以理解为"处理的字数"。在 `sessions` 列表中可以看到每条会话消耗的 token 总数，这有助于了解使用量。

### 4.4 对话突然中断了，数据会丢吗？

不会。每条消息发送后都会被立即保存到本地数据库。即使程序意外退出（比如关掉了终端窗口），最多丢失当前正在生成的这条回复，此前的全部对话记录在下次启动时都可以恢复。

### 4.5 换模型后之前的聊天记录还在吗？

在同一个会话中切换模型，聊天记录会保留。程序会带着已有的历史上下文，用新模型继续回复。你可以用 `sessions` 查看某次会话使用的模型，但在一个会话内换过模型后，列表里只会显示当前模型。

### 4.6 怎么查看当前对话用的是哪个模型和预设？

在 `chat` 命令创建新会话时，程序会显示你选择的模型名。如果想中途查看：
- 退出对话回到主菜单，输入 `sessions` 查看当前活跃会话的模型名。
- 关于预设：创建会话时选择的预设会在创建那一刻被应用（设定 AI 的角色指令），之后会话中不再显示预设名称。

## 5. 数据与隐私

### 5.1 数据存在哪里

所有数据（用户信息、对话记录、预设配置等）存储在项目目录的 `data/sqlite/app.db` 文件中。这是一个 SQLite 数据库文件，不依赖外部数据库服务。导出的 Markdown 文件保存在 `data/users/<用户名>/exports/` 目录下。日志文件保存在 `data/logs/` 目录下。

### 5.2 如何备份

需要备份的数据只有 `data/sqlite/app.db` 这一个文件。定期复制这个文件到安全位置即可。恢复时按第 3.4 节的步骤操作。注意 `.env` 文件中的 API 密钥也需要单独保管，它不在 `data/` 目录下。
